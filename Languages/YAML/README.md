# YAML

[← Back to Languages](../README.md)

**Why in DevOps:** YAML is the default config format for **Kubernetes** manifests, **CI/CD** pipelines (GitHub Actions, GitLab CI, many others), **Ansible**, and many cloud and IaC tools. You’ll edit YAML daily when managing K8s, CI, and automation. Content is **not duplicated here** — it appears wherever those tools are documented.

**Where it’s used:** YAML shows up across the handbook:

- **[Cloud-Native](../Cloud-Native/README.md)** — Kubernetes manifests, Helm values, Kustomize.
- **[CiCd](../CiCd/README.md)** — Pipeline configs (GitHub Actions, GitLab CI, Jenkins declarative, etc.).
- **[IAC](../IAC/README.md)** — Ansible playbooks, some Terraform and cloud configs.
- **[Automation](../Automation/README.md)** — Ansible, other automation config.

Use this folder as a **signpost**: when you look under Languages for “YAML” or “YML,” you’re pointed to where it’s used. Add topic files here only if you want a short syntax/reference (e.g. YAML basics, indentation, anchors) that doesn’t belong in a single section.
