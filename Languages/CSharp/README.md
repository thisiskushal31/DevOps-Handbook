# C#

[← Back to Languages](../README.md)

**Why in DevOps and security:** **C#** and **.NET** are used in Azure tooling, PowerShell, Windows automation, Azure DevOps pipelines, Azure Functions, and many enterprise integrations. When you work in Microsoft-centric environments, knowing C# syntax, the .NET SDK, and NuGet helps with maintenance and security review. **Before you dive in:** Topic 1 explains **what this language does** and **why it's here**. The section is ordered **basics → advanced → implementation**: from program structure and types through OOP, async, and tooling to use cases and security.

**Use cases and implementation:** For where C# runs (Azure, web APIs, desktop, tooling) and how to build and deploy safely, see **[Topic 19 — Use cases and applications](./19_Use_Cases_And_Applications.md)** and **Topic 20** (security and best practices).

**How this section is organized:** (1) **Start from the very basic** — Topics 1–5 cover what C# is, environment setup, program structure, basic syntax, and data types. (2) **Build bit by bit** — Topics 6–12 add operators, control flow, methods, arrays and strings, classes and OOP, inheritance and polymorphism, and interfaces and enums. (3) **Exceptions, files, and advanced** — Topics 13–18 cover exception handling, file I/O, namespaces and assemblies, delegates and events, generics and LINQ, async/await, and NuGet and testing. (4) **Then where you can implement** — Topics 19–20 show use cases and security. Follow the topics in order; each builds on the previous.
