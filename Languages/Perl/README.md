## Basic understanding (from official docs)

Perl officially stands for **Practical Extraction and Report Language**. It was originally a language optimized for scanning arbitrary text files, extracting information from them, and printing reports. It quickly became a mainstay for system management tasks and has grown into a general-purpose programming language, used for everything from quick one-liners to full-scale applications. The language is intended to be practical (easy to use, efficient, complete) rather than minimal. It combines features of sed, awk, and sh, making it familiar for Unix users. It supports procedural, functional, and object-oriented programming. Perl's strengths in text processing remain: it has powerful regular expressions and strong Unicode support. The CPAN (Comprehensive Perl Archive Network) provides a large collection of modules for almost any task. For beginners, the docs recommend starting with `perldoc perlintro`; the manual is split into overview, tutorials, reference (operators, functions, variables, modules), and platform-specific sections.
