# Shell (Bash, sh, zsh)

[← Back to Languages](../README.md)

**Why in DevOps:** Shell is the default for ad-hoc scripting, cron jobs, and glue on Linux and macOS. Most “run this script” in docs is shell. Content is **not duplicated here** — it lives in Operating-Systems.

**Where it’s covered:** Shell syntax, scripting, and commands are covered in depth under Operating-Systems:

- **[Linux: Shell and scripting](../Operating-Systems/Linux/9_Shell_And_Scripting.md)** — Bash/zsh, redirection, variables, control flow, robustness, DevOps patterns.
- **[Unix: Shell and scripting](../Operating-Systems/Unix/9_Shell_And_Scripting.md)** — sh, ksh, csh, bash on Unix.
- **[Windows: PowerShell](../Operating-Systems/Windows/1_Windows_Commands_And_PowerShell.md)** — PowerShell as the shell/scripting layer on Windows.

Use this folder as a **signpost**: when you look under Languages for “shell”, you’re pointed to the right place. Add topic files here only if you want Language-specific notes (e.g. “when to use shell vs Python”) that don’t belong in Operating-Systems.
