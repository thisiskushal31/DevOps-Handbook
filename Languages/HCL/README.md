# HCL (HashiCorp Configuration Language)

[← Back to Languages](../README.md)

**Why in DevOps:** **Terraform** uses HCL (`.tf` files) for infrastructure-as-code. Packer and other HashiCorp tools also use HCL. It’s a DSL for declarative config, not a general-purpose language. Content is **not duplicated here** — it lives under IAC (Terraform).

**Where it’s covered:** HCL and Terraform are covered under Infrastructure as Code:

- **[IAC (Infrastructure as Code)](../IAC/README.md)** — IAC concepts, patterns, and tool index.
- **[Terraform](../IAC/Terraform/README.md)** — Terraform: HCL, providers, state, modules, workspaces, remote backends, and use in multi-cloud and pipelines.

Use this folder as a **signpost**: when you look under Languages for “HCL” or “Terraform config”, you’re pointed to IAC/Terraform. Add topic files here only if you want language-specific notes (e.g. HCL syntax quick reference) that don’t belong in IAC.
