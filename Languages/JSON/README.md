# JSON

[← Back to Languages](../README.md)

**Why in DevOps:** JSON is the common data format for APIs, config, and pipeline definitions. Used in CI configs, cloud APIs, Terraform state, package.json, and many tool outputs. Content is **not duplicated here** — it appears wherever config and APIs are used.

**Where it’s used:** JSON shows up across the handbook:

- **[CiCd](../CiCd/README.md)** — Pipeline and job configs, API payloads.
- **[IAC](../IAC/README.md)** — Terraform state, provider config, some IaC outputs.
- **[Cloud-Native](../Cloud-Native/README.md)** — K8s manifests (often YAML; JSON is valid too), API requests/responses.
- **Languages/JavaScript** — `package.json`, npm; many APIs return JSON.

Use this folder as a **signpost**: when you look under Languages for “JSON,” you’re pointed to where it’s used. Add topic files here only if you want a short syntax/reference (e.g. JSON vs YAML, validation) that doesn’t belong in a single section.
